# 🏥 อธิบายระบบร้านยา DBMS แบบง่ายๆ

> **สำหรับใครก็ตามที่ไม่คุ้น Database** — อ่านแล้วต้องเข้าใจทันที!

---

## 🗂️ ภาพรวม — ระบบนี้คืออะไร?

**เหมือนร้านยาจริงๆ** แต่อยู่ในคอมพิวเตอร์:
- มียา → เก็บใน **ตาราง (Table)**
- มีใบสั่งซื้อ → เก็บใน **ตาราง**
- มีบิลขาย → เก็บใน **ตาราง**
- มีพนักงาน → เก็บใน **ตาราง**

---

## 📦 Table คืออะไร?

ลองนึกถึง **สมุดบัญชี** หรือ **Excel**:

```
ตาราง Product (รายการยา)
┌─────────────┬──────────────────┬───────────┐
│ Product_ID  │ Product_Name     │ Unit_Price│
├─────────────┼──────────────────┼───────────┤
│ PRD0000001  │ ยาพาราเซตามอล   │ 5.00      │
│ PRD0000002  │ ยาแก้ไอ         │ 30.00     │
└─────────────┴──────────────────┴───────────┘
```

แต่ละ **แถว** = ยา 1 ชนิด  
แต่ละ **คอลัมน์** = ข้อมูล 1 อย่าง (ชื่อ, ราคา, ฯลฯ)

---

## 🔑 Primary Key (PK) — รหัสประจำตัว

เหมือน **เลขบัตรประชาชน** — ห้ามซ้ำ ห้ามว่าง

```
Product_ID = "PRD0000001"  ← ไม่มียาตัวไหนมี ID ซ้ำกันเด็ดขาด
```

**ทำไมต้องมี PK?** → เพราะยาอาจชื่อเหมือนกันได้ แต่ ID ต้องต่างกัน  
(เหมือนคนสองคนชื่อ "สมชาย" แต่เลข ID บัตรต่างกัน)

---

## 🔗 Foreign Key (FK) — การเชื่อมโยงตาราง

ลองนึกถึง **ใบสั่งซื้อในชีวิตจริง**:

```
ใบสั่งซื้อ → ระบุว่า: ซื้อจาก "ร้านยา ABC" (Supplier_ID = "S01")
```

`Supplier_ID` ใน `Purchase_Header` → ต้องมีอยู่จริงใน `Supplier` table  
**ถ้าใส่ Supplier ที่ไม่มี → Oracle ปฏิเสธทันที** (ORA-02291)

```
เหมือน: สั่งซื้อจากร้านที่ไม่มีในสมุดรายชื่อ → ระบบไม่ยอม
```

---

## 🔄 Stored Procedure — สูตรทำอาหารสำเร็จรูป

**Stored Procedure** เหมือน **สูตรทำอาหาร** ที่บันทึกไว้ล่วงหน้า

```
สูตร: "ขายยา" (sp_add_sale_item)
1. เช็คว่ายาเหลือพอไหม?
2. ถ้าพอ → บันทึกการขาย
3. หักสต็อกออก
4. บวกยอดเงินในบิล
```

**ข้อดี**: ทุกคนที่ขายยาใช้ "สูตร" เดียวกัน → ไม่มีข้ามขั้นตอน ไม่มีเอาใจช่วย

---

## ⚛️ Transaction — "ต้องสำเร็จพร้อมกัน หรือล้มทั้งหมด"

**อุปมา**: โอนเงินผ่านธนาคาร

```
ขั้นที่ 1: หักเงินจากบัญชี A  ✅
ขั้นที่ 2: เพิ่มเงินเข้าบัญชี B  ❌ (ระบบล่ม!)
```

ถ้าไม่มี Transaction → เงินหายไปเฉยๆ!  
ถ้ามี Transaction → ขั้นที่ 1 **ถูกยกเลิกอัตโนมัติ** (Rollback) → เงินกลับมา

**ในโปรเจคนี้**:
```
ขั้นที่ 1: สร้างบิลขาย           ✅
ขั้นที่ 2: ตัดสต็อกยาตัวที่ 1   ✅
ขั้นที่ 3: ตัดสต็อกยาตัวที่ 2   ❌ (สต็อกหมด)

→ ROLLBACK: ยกเลิกทั้งหมด บิลหาย สต็อกคืนกลับ
```

**COMMIT** = กดยืนยัน "บันทึกแน่นอน"  
**ROLLBACK** = กด Ctrl+Z "ยกเลิกทั้งหมด"

---

## 🔒 SELECT FOR UPDATE — จองคิว

**อุปมา**: ที่นั่งในโรงหนัง

```
คนที่ 1: ดูว่าที่นั่ง A1 ว่างไหม? → ว่าง!
คนที่ 2: ดูว่าที่นั่ง A1 ว่างไหม? → ว่าง!
คนที่ 1: จอง A1
คนที่ 2: จอง A1  ← ปัญหา! 2 คนได้ที่นั่งเดียวกัน
```

`SELECT FOR UPDATE` = **ล็อกที่นั่งทันทีที่ดู** → คนที่ 2 ต้องรอ

```sql
SELECT Remaining_Qty FROM Product_Batches WHERE Batch_ID = 'B001'
FOR UPDATE;  -- ← ล็อกแถวนี้ไว้ก่อน ใครจะแก้ต้องรอ
```

## 🤖 Trigger — พนักงานอัตโนมัติที่คอยดูอยู่

**Trigger** เหมือน **กล้องวงจรปิด + แจ้งเตือนอัตโนมัติ**

```
trg_product_audit (กล้องวงจรปิด):
- ทุกครั้งที่แก้ไขสินค้า → บันทึกว่าใครแก้ เมื่อไร แก้อะไร

trg_low_stock_alert (สัญญาณเตือน):
- ทุกครั้งที่สต็อกลดลง → ถ้าเหลือน้อยกว่ากำหนด → บันทึก Alert
```

**ข้อดี**: ไม่ต้องเขียนโค้ดซ้ำทุกที่ → Trigger ทำงานเองอัตโนมัติ

---

## 👁️ View — หน้าต่างมองข้อมูล

**View** เหมือน **รายงานสำเร็จรูป** ที่กดปุ่มแล้วได้เลย

```
v_stock_summary = รายงานสต็อกยาทั้งหมด
→ ข้างในซับซ้อนมาก (JOIN 3 ตาราง + SUM + CASE)
→ แต่เราแค่พิมพ์: SELECT * FROM v_stock_summary
   แล้วได้รายงานสวยงามทันที
```

เหมือนกด **"สรุปยอด"** ใน Excel แทนที่จะคำนวณเอง

---

## 🔢 Sequence — เลขคิว

**Sequence** เหมือน **ตู้จ่ายบัตรคิว** ที่ธนาคาร

```sql
seq_sale.NEXTVAL  -- กด "รับบัตรคิว" → ได้เลข 101
seq_sale.NEXTVAL  -- กดอีกที → ได้เลข 102
```

- ไม่มีวันซ้ำ แม้ 100 คนกดพร้อมกัน
- Sale_ID: `SAL0000000101`, `SAL0000000102`, ...

---

## 📜 Cursor — อ่านผลลัพธ์ทีละแถว

**Cursor** เหมือน **นิ้วที่ชี้อ่านหนังสือ** ทีละบรรทัด

```sql
-- แทนที่จะเอาผลลัพธ์ทั้งหมดมาพร้อมกัน
-- Cursor อ่านทีละแถว แถวต่อแถว
FOR rec IN (SELECT * FROM Purchase_Detail WHERE Purchase_ID = 'P001') LOOP
    -- rec คือ "แถวปัจจุบัน" ที่นิ้วกำลังชี้อยู่
    INSERT INTO Product_Batches VALUES (rec.Product_ID, rec.Order_Qty, ...);
END LOOP;
```

**ใช้เมื่อ**: ต้องทำอะไรบางอย่างกับ **ทุกแถว** ในผลลัพธ์

## 🛡️ Bind Variable — ป้องกันแฮกเกอร์

**อุปมา**: กรอกฟอร์มกับพูดปาก

```
❌ พูดตรงๆ (อันตราย):
   "ค้นหายาชื่อ: ' OR 1=1 --"
   → แฮกเกอร์ส่งคำสั่งปลอมเข้าระบบ!

✅ กรอกฟอร์ม (ปลอดภัย):
   ช่อง "ชื่อยา": [' OR 1=1 --]
   → ระบบถือว่าเป็นแค่ "ข้อความ" ไม่ใช่คำสั่ง
```

```js
// ❌ อันตราย
`SELECT * FROM Product WHERE name = '${userInput}'`

// ✅ ปลอดภัย
`SELECT * FROM Product WHERE name = :name`
{ name: userInput }  // ส่งแยก → Oracle รู้ว่าเป็นข้อมูลไม่ใช่ SQL
```

## 🏊 Connection Pool — สระว่ายน้ำ Connection

**อุปมา**: การ์ดเข้าออฟฟิศ

```
❌ ไม่มี Pool: ทุกคนต้องทำการ์ดใหม่ทุกครั้งที่เข้าออฟฟิศ (เสียเวลา)
✅ มี Pool:   มีการ์ดสำรอง 10 ใบ → หยิบใช้ → คืนเมื่อเสร็จ
```

```
Pool มีการ์ด 10 ใบ (max=10)
Request ที่ 1 → หยิบการ์ดใบที่ 1 → ทำงาน → คืน
Request ที่ 2 → หยิบการ์ดใบที่ 1 (ที่เพิ่งคืน) → ทำงาน → คืน
```

**สำคัญมาก**: ถ้าลืมคืนการ์ด → การ์ดหมด → ทุกคนเข้าออฟฟิศไม่ได้ → ระบบล่ม!

## 🔍 JOIN — เปิดสมุดหลายเล่มพร้อมกัน

**อุปมา**: ดูข้อมูลพนักงาน + แผนกพร้อมกัน

```
สมุดที่ 1 (Sales_Header): บิลขาย → รู้ว่า EMP_ID = "E001"
สมุดที่ 2 (Employees):    รหัส "E001" = "นายสมชาย ใจดี"
```

```sql
SELECT sh.Sale_ID, e.First_Name || ' ' || e.Last_Name AS ชื่อพนักงาน
FROM Sales_Header sh
LEFT JOIN Employees e ON sh.EMP_ID = e.EMP_ID
```

**LEFT JOIN**: เอาทุกบิล แม้บางบิลไม่รู้ว่าใครขาย (NULL)  
**INNER JOIN**: เอาเฉพาะบิลที่รู้ว่าใครขาย

## 📊 GROUP BY + Aggregate — สรุปเป็นกลุ่ม

**อุปมา**: สรุปยอดขายแต่ละเดือน

```
ข้อมูลดิบ (บิลทั้งหมด):
บิล 001 - มกราคม - 500 บาท
บิล 002 - มกราคม - 300 บาท
บิล 003 - กุมภาพันธ์ - 200 บาท

GROUP BY เดือน + SUM(ยอด):
มกราคม   → 800 บาท  (500+300)
กุมภาพันธ์ → 200 บาท
```

```sql
SELECT TO_CHAR(Sale_date, 'YYYY-MM') AS เดือน,
       COUNT(*)           AS จำนวนบิล,
       SUM(Total_Amount)  AS ยอดรวม
FROM Sales_Header
GROUP BY TO_CHAR(Sale_date, 'YYYY-MM')
```

## 🚨 Exception Handling — แผนสำรองเมื่อพัง

**อุปมา**: ร้านอาหารที่เตรียมแผนรับมือ

```
ปกติ: ลูกค้าสั่ง → เสิร์ฟ
เกิดเหตุการณ์ผิดปกติ:
  - ของหมด → "ขออภัย ของหมดแล้ว"
  - ไฟดับ  → "ขออภัย ระบบขัดข้อง"
  - อื่นๆ  → "ขออภัย เกิดข้อผิดพลาด: [รายละเอียด]"
```

```sql
BEGIN
    -- ทำงานปกติ
    SELECT Remaining_Qty INTO v_stock FROM Product_Batches ...;
EXCEPTION
    WHEN NO_DATA_FOUND THEN  -- ของหมด
        p_result := 'ERROR: ไม่พบสินค้า';
    WHEN OTHERS THEN         -- เหตุการณ์อื่น
        p_result := 'ERROR: ' || SQLERRM;
END;
```

## 🧩 CASE WHEN — ถ้า-แล้ว ใน SQL

**เหมือน If-Else แต่อยู่ใน SQL**

```sql
CASE 
    WHEN สต็อก = 0              THEN 'หมดสต็อก'  -- ถ้า...แล้ว...
    WHEN สต็อก < Reorder_Point  THEN 'สต็อกต่ำ'
    ELSE                              'ปกติ'
END AS สถานะสต็อก
```

เหมือนป้ายที่ระบบติดให้ยาแต่ละตัว: 🔴หมดสต็อก / 🟡สต็อกต่ำ / 🟢ปกติ

## 🗺️ ภาพรวม Data Flow — ตอนกดขาย เกิดอะไร?

```
1. ลูกค้าซื้อยา
       ↓
2. พนักงานกดชำระเงินในเว็บ (app.js)
       ↓
3. เว็บส่งข้อมูลไปยัง Server (sales.js)
       ↓
4. Server ขอ Connection จาก Pool
       ↓
5. เรียก sp_create_sale → สร้างบิล (SAL0000000101)
       ↓
6. เรียก sp_add_sale_item สำหรับแต่ละยา:
   - ล็อกสต็อก (FOR UPDATE)
   - เช็คว่าพอ
   - บันทึกรายการขาย
   - หักสต็อก
   - บวกยอดบิล
   - Trigger เช็คสต็อกต่ำอัตโนมัติ
       ↓
7. ทุกอย่างสำเร็จ → COMMIT (บันทึกถาวร)
   มีอะไรพัง → ROLLBACK (ยกเลิกทั้งหมด)
       ↓
8. คืน Connection กลับ Pool
       ↓
9. เว็บแสดงใบเสร็จ 🎉
```

## 📝 สรุปคำศัพท์สำคัญ

| คำ | ความหมายง่ายๆ |
|----|--------------|
| **Table** | ตาราง เหมือน Excel Sheet |
| **Primary Key** | เลข ID ห้ามซ้ำ เหมือนบัตร ปชช. |
| **Foreign Key** | อ้างอิงไปยังตารางอื่น เหมือนลิงก์ |
| **Stored Procedure** | สูตรสำเร็จรูปใน Database |
| **Transaction** | กลุ่มคำสั่งที่ต้องสำเร็จพร้อมกัน |
| **COMMIT** | กดบันทึกถาวร |
| **ROLLBACK** | กด Ctrl+Z ยกเลิกทั้งหมด |
| **Trigger** | สัญญาณเตือนอัตโนมัติเมื่อเกิดเหตุ |
| **View** | รายงานสำเร็จรูป |
| **Sequence** | ตู้จ่ายบัตรคิว |
| **Cursor** | นิ้วชี้อ่านผลลัพธ์ทีละแถว |
| **JOIN** | เปิดสมุดหลายเล่มพร้อมกัน |
| **GROUP BY** | จัดกลุ่มแล้วสรุป |
| **Bind Variable** | กรอกฟอร์ม (ปลอดภัย ≠ พูดปาก) |
| **Connection Pool** | ที่ล็อกเกอร์การ์ดสำรอง |
| **FOR UPDATE** | จองคิว ไม่ให้คนอื่นเข้ามาพร้อมกัน |
| **Index** | สารบัญหนังสือ ค้นเจอไว |
| **NULL** | ไม่มีค่า (≠ ศูนย์ ≠ ว่าง) |
| **NVL** | ถ้า NULL ให้ใช้ค่านี้แทน |
